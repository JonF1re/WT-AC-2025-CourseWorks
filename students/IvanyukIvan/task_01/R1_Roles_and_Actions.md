# Вариант 14 — Прогресс «До сессии успею» — Роли и действия

Краткое описание

MVP: Topic, Goal, ProgressEntry, User. UI отображает диаграммы прогресса и список целей; API: /topics, /goals, /progress, /reports. Приёмка (MVP): корректное создание и отображение диаграмм прогресса.

Роли и основные действия

- Администратор
  - CRUD для пользователей
  - CRUD для тем (Topic) и целей (Goal)
  - Просмотр прогресса всех пользователей
  - Просмотр отчётов и статистики

- Пользователь
  - Просмотр списка своих тем и целей
  - Добавление записей прогресса (ProgressEntry)
  - Просмотр личных диаграмм и графиков
  - Генерация личных отчётов
  - Настройка напоминаний (опционально)

Примеры сценариев

- Администратор создаёт Topic и Goal, назначает их пользователям → пользователь видит цели и начинает отмечать прогресс.
- Пользователь добавляет ProgressEntry, отмечая выполнение задачи → система обновляет диаграммы и показывает процент выполнения цели.
